进制简介、转换与运算

一点进制知识总结

1. 进制简介

进制：即进位制，是我们规定的一种进位方式，表示某一位置上的数，运算时是逢几进一位。十进制逢十进一，十六进制逢十六进一……

为什么要学习进制？

计算机数据在底层运算时候，都以二进制形式运作，也有数据以八进制、十进制、或者十六进制进行存储或运算，了解不同的进制，有助于理解数据的运算过程。

二进制（0b）（binary）

只用 0 和 1 两个数码来表示。进位规则：逢二进一，借位规则是 “借一当二”。

八进制（0）（octal）
十六进制（0x）（hex）

用数字 0 到 9 和字母 A 到 F（或 a-f）表示，其中：A — F 表示10 - 15，这称为十六进制。

2.进制转换

几进制转十进制

系数 * 基数的权次幂相加

  • 系数：每一【位】上的数
  • 基数：几进制，就是几
  • 权：从数值的右侧，以 0 开始，逐个 + 1增加。
十进制到二进制的转换

除基取余

使用源数据，不断的除以基数（几进制，基数就是几）得到余数，直到商为零，再将余数倒着拼起来。

快速进制转换（8421码）：

  • 8421 码

    8421 码又称为 BCD 码

    BCD：（Binary-Coded Decimal）二进制码十进制数

    在这种编码方式中，每一位二进制值的 1 都是代表一个固定数值，把每一位的 1 代表的十进制数加起来得到的结果就是它所代表的十进制数。

    1 1 1 1 1 1 1 1
    128 64 32 16 8 4 2 1

  • 0 所对应的值不取， 1 所对应的值取出并相加。

二进制快速转八进制

八进制：将三个二进制位看为一组，在进行转换

二进制快速转十六进制

十六进制：将四个二进制位看为一组，再进行转换

3.原码、反码和补码

原码： 二进制码，最高位 0 表示正 1 表示负

反码： 正数不变，负数符号位不变，其他位取反

补码： 正数不变，负数 +1；

4.位运算符

** & | ~ ^ **

  • & 并且 有false则false， 也就是 0 则 0;

  • ```java
    // 规律： 任何数和0去与 都是0;
    //0
    System.out.println(100&0);
    System.out.println(13127128&0);

  • ~ 取反

  • ^ 异或 不同为true 1 相同为false 0

  • 两个数据的交换

  • 位移：<< 左移

  • >> 有符号右移

  • >>> 无符号右移

  • 冒泡排序

  • 矩阵转置

修改 Windows 控制台的默认代码页

注意：不同版本或计算机都有可能导致修改方式不同，本文仅供参考。

1. 为什么要修改这玩意

一般在程序开发中你使用的编码是统一的 UTF-8，在 Windows 下有时会遇到乱码问题，比如下面这个程序：

运行时会出现中文乱码。

这是因为 Window cmd 的默认编码是 GBK。与程序采用的 UTF-8 不一致造成的中文及特殊字符乱码。

查看系统编码

打开运行 —— 命令提示符窗口输入 chcp，回车。

或者，打开 cmd 后，点击 cmd 窗口左上角图标，选”属性”菜单。

修改方法

第一种：临时修改编码

使用 chcp 命令，例如 chcp 65001，这会将当前代码页变为 utf-8 编码，不过这种方式在关闭 cmd 之后会自动失效。

十进制码值 对应编码名称
950 繁体中文
65001 UTF-8 代码页
936 简体中文默认的 GBK
437 MS-DOS 美国英语
第二种：永久性修改

永久修改及修改注册表

打开注册表编辑器：运行 —— 输入 regedit。

定位到：HKEY_CURRENT_USER\Console%SystemRoot%_system32_cmd.exe

将 CodePage 值修改为 十进制 65001 。

或

转到 [HKEY_LOCAL_MACHINE\Software\Microsoft\Command Processor\Autorun]

将值修改为 @chcp 65001>nul

或

转到 [HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Control\Nls\CodePage\OEMCP]

将值修改为65001

如果改完仍无法正常显示 UTF-8 字符，则需改变字体属性：在命令行标题栏上点击右键，选择”属性” -> “字体”，将字体修改为 True Type 字体 “Lucida Console” ，然后点击确定将属性应用到当前窗口。

不得不提的事

不懂 Windows 操作系统的内部运作

一开始进入注册表编辑器时在 HKEY_CURRENT_USER\Console\

下没有 %SystemRoot%_system32_cmd.exe

只有关于 PowerShell 的设置，过了一段时间，

正找其它教程时，再次打开注册表编辑器，

发现多出了两个目录，一个关于 cmd ，一个关于 CodeBlocks。

然后，只是修改 cmd 并不能完全解决开头的代码输出问题，最后修改 CodeBlocks 的 CodePage 值为 437 才得到理想的输出。
